## <a id="features"></a>✨ Возможности

`agy-manager` объединяет в себе два ключевых инструмента для удобной разработки в экосистеме Antigravity:

- 🔓 **Обход региональных ограничений:** Отключение локальных заглушек доступности ("not available in your location") во всех трех ключевых приложениях (CLI, Manager, IDE).
- 👥 **Менеджер аккаунтов:** Безопасное сохранение и быстрое переключение между несколькими профилями авторизации без необходимости повторного входа через браузер.
- 🎨 **Интерактивный TUI:** Удобное консольное меню для управления патчами и аккаунтами на базе библиотек `rich` и `questionary`.
- ⚡ **Ядро без зависимостей:** Все основные консольные команды работают на стандартной библиотеке Python без необходимости установки сторонних пакетов.
- 🛡️ **Безопасность и откат:** Автоматическое резервное копирование (`*.agybak`) файлов перед модификацией для безопасного отката изменений в один клик.
- ⚙️ **Умный автопоиск:** Автоматическое сканирование реестра Windows, системного PATH, переменных окружения и путей Scoop для динамического обнаружения установленных приложений.
- 🧬 **Устойчивость к обновлениям:** Поиск уникальных сигнатур и инструкций через регулярные выражения вместо привязки к статическим файловым смещениям.

## <a id="quick-start"></a>🚀 Быстрый старт

### Вариант А: Интерактивный TUI (рекомендуется)

Запуск полноценного интерактивного интерфейса с таблицей статусов для управления как патчами, так и аккаунтами:

1. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```
2. **Запустите agy-manager:**
   ```bash
   python manager.py
   ```

*(Интерфейс поддерживает навигацию стрелочками, выбор пробелом и подтверждение по Enter).*

---

### Вариант Б: Консольный CLI (без зависимостей)

Работает полностью на стандартной библиотеке Python (установка сторонних пакетов не требуется). Позволяет автоматизировать действия и управлять окружением напрямую из терминала.

| Команда | Описание |
| :--- | :--- |
| `python manager.py status` | Проверить статус патча для всех приложений. |
| `python manager.py patch` | Пропатчить все найденные приложения. |
| `python manager.py restore` | Восстановить все приложения из оригинальных бэкапов. |
| `python manager.py patch <cli\|manager\|ide>` | Пропатчить только выбранные приложения. |
| `python manager.py accounts <cli-manager\|ide> <действие> [имя1] [имя2]` | Управление сохраненными профилями авторизации (подробнее см. ниже). |

> [!TIP]
> Если ваше приложение установлено в нестандартную директорию, вы можете вручную указать путь к нему:
> ```bash
> python manager.py --path-cli "D:\CustomTools\agy.exe" patch cli
> ```

## <a id="bypass"></a>🔓 Патчинг региональных ограничений

Патчер модифицирует локальные проверки доступности, позволяя приложениям функционировать в регионах с ограничениями.

> [!NOTE]
> Эти экраны — чисто локальное ограничение на стороне клиента. После их обхода модели и функции работают в штатном режиме. Этот инструмент **не** обходит серверную авторизацию и не открывает доступ к платным функциям, он лишь отключает локальную заглушку региональных ограничений.

### Поддерживаемые приложения

| Идентификатор | Приложение | Метод патчинга | Маркер поиска |
| :---: | :--- | :--- | :--- |
| **`cli`** | **Antigravity CLI** (`agy.exe`) | Бинарный патч `agy.exe`, нейтрализующий проверку гейта `hasValidAuth`. | `agy.exe` |
| **`manager`** | **Antigravity Manager** (Electron) | Бинарный патч Go-бэкенда `language_server.exe` для принудительной установки флага `hasValidAuth` в `true`. | `resources\bin\language_server.exe` |
| **`ide`** | **Antigravity IDE** (VS Code) | Патч минифицированного скрипта запуска VS Code для принудительного включения флага `isGoogleInternal`. | `resources\app\out\main.js` |

## <a id="accounts"></a>👥 Менеджер профилей авторизации

Позволяет сохранять текущую сессию авторизации Antigravity под уникальным именем (профилем) и переключаться между ними в офлайне без повторного входа в браузере. 

### Области управления
Управление сессиями разделено на две независимые области:
1. **CLI + Manager** (используют общий токен доступа в Windows Credential Manager).
2. **IDE** (использует собственные ключи авторизации в SQLite базе данных `state.vscdb` внутри VS Code).

Это разделение позволяет независимо переключать профили для разных инструментов и избегает конфликтов блокировки файлов.

### Использование в интерактивном меню:
1. Выберите пункт **Manage accounts** в главном меню `python manager.py`.
2. Выберите целевую область: **CLI + Manager** или **IDE**.
3. Доступны операции сохранения текущей сессии, переключения на сохраненный профиль, удаления профиля или выхода.

### Использование в командной строке:
Формат команды: `python manager.py accounts <cli-manager|ide> <действие> [имя]`

| Действие | Пример команды | Описание |
| :--- | :--- | :--- |
| `list` (или `ls`) | `python manager.py accounts cli-manager list` | Список сохраненных профилей для выбранной области с отметкой активного. |
| `save <имя>` | `python manager.py accounts cli-manager save work` | Сохранить текущую активную сессию под указанным именем. |
| `use <имя>` (или `switch`) | `python manager.py accounts cli-manager use personal` | Переключиться на сохраненный профиль. |
| `logout` | `python manager.py accounts cli-manager logout` | Выйти из текущей сессии локально (чтобы войти в другой аккаунт). |
| `rename <старое> <новое>` (или `mv`) | `python manager.py accounts cli-manager rename work personal` | Переименовать сохраненный профиль. |
| `rm <имя>` | `python manager.py accounts cli-manager rm work` | Удалить сохраненный профиль. |
| `current` (или `who`) | `python manager.py accounts cli-manager current` | Показать имя текущего активного профиля. |

## <a id="details"></a>🔍 Детали реализации (под капотом)

<details>
<summary>🛠️ <b>Патч CLI (`agy.exe` Go-бинарник)</b></summary>

CLI выводит при запуске косметическую секцию «Eligibility Check». Решение о её показе принимается в функции `handleAuthResult`, которая читает поле `hasValidAuth` (байт по смещению `+8`) из AuthResult, пришедшего с сервера.

1. Патчер сканирует бинарник в поисках уникальной сигнатуры проверки гейта: `test rax,rax` → `je` (eligible) → `cmp byte ptr [rax+8], 0` → `jne` (eligible).
2. При нулевом `hasValidAuth` выполнение проваливается дальше и печатает ошибку.
3. Патч заменяет `cmp byte ptr [rax+8], 0` на `test rax,rax` (+`NOP`). Так как `rax` здесь гарантированно не равен нулю, переход `jne` всегда уводит выполнение в ветку «eligible».
</details>

<details>
<summary>📦 <b>Патч Manager (`language_server.exe` Go-бэкенд)</b></summary>

Менеджер общается с локальным Go-бэкендом `language_server.exe` по протоколу connect-rpc. Вердикт `hasValidAuth` (байт по смещению `+8` в AuthResult) определяется в единственном корневом месте — функции `authclient.(*PersonalAuthValidator).Validate`.

1. Патчер ищет в валидаторе сигнатуру проверки: `cmp byte ptr [rax+8], 0` → `je` (уводит выполнение в обход привязки токена).
2. Проверка вместе с переходом перезаписывается на `mov byte ptr [rax+8], 1` + `NOP`: флаг принудительно выставляется в `true`, а нейтрализованный `je` гарантирует, что выполнение всегда идёт в ветку привязки и сохранения токена.
3. Именно результат этого валидатора возвращает `GetAuthStatus` и на него же опирается функция логина, поэтому одного патча достаточно для всех сценариев — и первого входа, и последующих перезапусков. Токен сохраняется на диске, а экран ошибки не отображается.

> **Linux / macOS.** Патч `manager` кросс-платформенный. Go-бэкенд собран из одного исходника, поэтому машинный код на `linux/amd64` и `windows/amd64` идентичен — та же сигнатура и та же замена байтов применяются к ELF-бинарнику `resources/bin/language_server` (без `.exe`). Автопоиск для `manager` вне Windows сканирует стандартные префиксы установки (`/opt`, `/usr/share`, `/usr/lib`, `~/.local/share`, `/Applications`, каталог лаунчера `antigravity` из `PATH`); для нестандартного расположения укажите путь вручную через `--path-manager`. Перед патчем закройте приложение (иначе ОС вернёт `ETXTBSY`). Патчи `cli` и `ide`, а также управление аккаунтами (`accounts`) пока остаются только для Windows.
</details>

<details>
<summary>💻 <b>Патч IDE (`main.js` VS Code)</b></summary>

1. Патчер находит файл `resources/app/out/main.js` и с помощью регулярного выражения ищет минифицированный шаблон авторизации: `resetIsTierGCPTos\(\),this\.[A-Za-z_\$0-9]+\.isGoogleInternal`.
2. Заменяет его на `resetIsTierGCPTos(),true`, что принудительно включает флаг внутреннего сотрудника Google.
3. Очищает кэши байт-кода VS Code (`CachedData` и `Code Cache/js`) для моментального применения изменений.
</details>

<details>
<summary>👥 <b>Менеджер аккаунтов (хранение и переключение сессий)</b></summary>

Механизм переключения аккаунтов работает полностью локально и офлайн, не обращаясь к серверам авторизации (штатный выход из приложения инвалидирует токен на стороне сервера).

1. **Разделение хранилищ:** Токены CLI/Manager берутся из Windows Credential Manager (`gemini:antigravity`). Токены IDE вычитываются из SQLite базы данных VS Code `state.vscdb` (ключи `antigravityUnifiedStateSync.*`).
2. **Безопасное хранение:** При сохранении профиля данные шифруются/кодируются и записываются обратно в Credential Manager под уникальным именем с префиксом `agy-manager:account:cli-manager:<name>` или `agy-manager:account:ide:<name>`.
3. **Обход лимитов размера:** generic-записи в Credential Manager ограничены размером в 2560 байт, в то время как JSON-сессия IDE может превышать 8 КБ. Профили IDE автоматически шардируются на части по 2000 байт и сохраняются в виде пронумерованных записей (`.../<index>`).
4. **Синхронизация и защита от перезаписи:** Перед применением нового профиля текущий активный профиль автоматически синхронизируется, чтобы сохранить любые ротированные ключи сессии.
</details>

## <a id="warnings"></a>⚠️ Важные примечания

- **Совместимость версий:** Работоспособность патчера гарантируется только на **актуальных** версиях приложений. Он завязан на бинарные сигнатуры конкретных сборок, поэтому на устаревших версиях может не находить нужные инструкции и просто ничего не делать — статус отобразится как `unknown`, а файл останется нетронутым (безопасный отказ). В этом случае обновите приложение до последней версии.
- **Сброс при обновлениях:** При обновлении приложений пропатченные файлы будут перезаписаны оригинальными. Достаточно повторно запустить `python manager.py patch`.
- **Блокировка файлов и процессов:** Перед патчингом или переключением аккаунтов обязательно закройте все целевые приложения соответствующей области (CLI, Manager или IDE). В противном случае операционная система заблокирует доступ к файлам, либо запущенное приложение перезапишет восстановленные токены из своего кэша в оперативной памяти.
- **Безопасность токенов:** Все ваши учетные данные (как текущие активные, так и сохраненные профили) остаются строго на вашей машине. Они хранятся в защищенном системном хранилище Windows Credential Manager и SQLite базе данных VS Code и никогда не передаются сторонним серверам.
- **Условия использования:** Изменение проприетарного кода приложений может нарушать условия использования (ToS). Данный проект создан исключительно в образовательных целях — используйте его под свою ответственность.
